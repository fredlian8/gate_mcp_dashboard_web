主表格

查看指标说明（价格变化 / OI 变化 / 市场信号）
价格变化%（随上方时间框架）
取该时间框架最近 lookback=6 根K线：(Close_last - Close_prevN) / Close_prevN × 100%（prevN 为倒数第 7 根收盘）。
OI 变化%（随上方时间框架）
取 Gate 合约统计 contract_stats 最近 lookback=6 个同时间框架数据：(OI_last - OI_prevN) / OI_prevN × 100%（prevN 为倒数第 7 个点）。
市场信号（四象限）
由 价格变化% 与 OI 变化% 的符号组合判断：
1) 价格↑ + OI↑：多头强势进场（上涨伴随增仓）。
2) 价格↑ + OI↓：多头获利了结（上涨但减仓）。
3) 价格↓ + OI↑：空头强势进场（下跌伴随增仓）。
4) 价格↓ + OI↓：空头获利了结（下跌但减仓）。
注意事项
1) “最新价格”来自 K线最后收盘价（该时间框架），不等同于实时 ticker last。
2) contract_stats 与 K线数据更新时间可能不同步，极短时间内会有轻微偏差。

多空综合雷达（Signal Dashboard）

综合资金费率 / OI / 量能 / MACD / RSI 打分，输出强多/强空信号，并可配合 Telegram 推送。

score_raw updated_ts 市值排名 合约 分数 级别 价格 价格(tf)% 资金费率 OI(tf)% 量能(tf/24h) MACD RSI(14) 主要原因 更新时间
查看指标计算公式与打分规则
信号等级（score ∈ [-10, +10]）
strong_long：score ≥ +6；long：score ≥ +3；neutral：(-3, +3)；short：score ≤ -3；strong_short：score ≤ -6。
时间框架（K线选择）
本页面所有与 K线相关的指标都会跟随上方 K线 选择（15m/1h/1d），默认 15m。
价格涨跌（价格(tf)%）
使用 Gate 永续合约 K线：pct_tf = (Close_last - Close_prev) / Close_prev × 100%（取最新两根 tf K线的收盘价）。
OI 变化（OI(tf)%）
使用 Gate 永续合约 contract_stats：oi_change_tf = (OI_last - OI_prev) / OI_prev × 100%（取最近两条同 tf 数据）。 说明：OI(tf) 与上方 K线时间框架保持一致（15m/1h/1d）。
阈值与配置来源（可选A）
目前评分阈值（资金费率、共振、量能、MACD/RSI 加减分、强信号阈值）写在后端 main.py::_signal_score 中，为固定规则。 可配置项主要通过环境变量控制： SIGNAL_DASHBOARD_ENABLED（开关）、 SIGNAL_DASHBOARD_TOPN（TopN）、 SIGNAL_DASHBOARD_WATCHLIST（自选）、 SIGNAL_DASHBOARD_CACHE_TTL_SEC（缓存）、 SIGNAL_PUSH_ENABLED/SIGNAL_PUSH_INTERVAL_SEC/SIGNAL_PUSH_SCORE_STRONG/SIGNAL_PUSH_COOLDOWN_SEC（推送相关）。 如果你希望“各项打分阈值”也可配置，我可以把这些阈值抽成 env 或一个 JSON 配置。
资金费率（funding）
来源：Gate 永续 funding_rate。打分：
funding ≤ -0.03%：+2（需 OI(tf) 幅度确认，否则降档为 +1）；funding ≤ -0.01%：+1；funding ≥ +0.03%：-2（需 OI(tf) 幅度确认，否则降档为 -1）；funding ≥ +0.01%：-1。
OI(tf) 幅度确认阈值会随 tf 切换：15m≥1.0%，1h≥2.0%，1d≥5.0%（取绝对值）。
价格 × OI 共振（趋势确认）
含义：把 OI(tf)% 与 价格(tf)% 同时作为“趋势推进/趋势衰减”的判据。 只有当两者都达到阈值时才计入“强共振”，并按方向给出加减分。
阈值会跟随上方 K线时间框架（tf）自动切换： 15m：OI≥1.0%、价格≥0.3%；1h：OI≥2.0%、价格≥0.5%；1d：OI≥5.0%、价格≥1.5%。
打分（四种组合，均要求 |OI(tf)%| 与 |价格(tf)%| 同时过阈值）： OI↑ 且 价格↑：+3；OI↑ 且 价格↓：-3；OI↓ 且 价格↑：+1.5；OI↓ 且 价格↓：-1.5。
说明：当出现 OI↑ 的两种组合（增仓共振）时，后端会标记为“趋势确认”，并对 RSI 加减分自动降权，避免强趋势里 RSI 逆势拉分。
量能（量能(tf/24h)）
近似衡量“这根 tf K线成交量相对 24h 均值是否放大”： vol_ratio = V_tf / (V_24h / buckets)，其中 buckets = 1440 / tf_minutes。 默认使用“上一根已收盘”的 tf K线成交量（避免进行中 K线导致的小时内抖动）。 打分（方向敏感）：vol_ratio ≥ 2.5 时 |Δ|=2.5，vol_ratio ≥ 1.5 时 |Δ|=1.5；若价格(tf)% > 0 视为“放量上涨”加分，若价格(tf)% < 0 视为“放量下跌”扣分。
MACD（随 tf）
使用 tf K线 close 序列计算 MACD(12,26,9)。表格中的 MACD 列展示“交叉/预警事件”：检测最近 3 根是否出现金叉/死叉（取最近一次交叉），以及“即将金叉/即将死叉”预警（偏早：距离接近 ratio≤0.75 + 连续收敛 lookback=2 + 同向斜率）。 打分：金叉 +3.5；即将金叉 +2；死叉 -3.5；即将死叉 -2。
RSI(14)（随 tf）
使用 tf K线 close 序列计算 RSI(14)。打分： RSI < 30：+2；RSI < 40：+1；RSI > 70：-2；RSI > 60：-1。
当出现趋势确认（MACD 金叉/死叉 或 OI↑ 的强共振）时，RSI 加减分会自动降权为 0.5 倍。
注意事项
1) score 会被裁剪到 [-10, +10]。 2) “当前价格”字段来自 ticker last（更实时），而涨跌%来自 K线收盘价变化（Close/PrevClose），两者口径不同属正常。 3) 数据存在缓存与接口刷新周期，短时间内可能出现轻微不同步。 4) Telegram 强信号推送默认使用 1h K线口径（可用环境变量 SIGNAL_PUSH_K_TF 修改），不一定等同于你在页面上当前选择的 K线时间框架。 5) 价格×OI 共振除了 OI↑ 组合外，也会对 OI↓（减仓）组合给予较小权重，以增强震荡/反弹场景下的解释性。 6) 当出现趋势确认（MACD 金叉/死叉 或 OI×价格强共振）时，RSI 的加减分会自动降权，避免在强趋势中逆势拉分。
提示：页面上的“只看有信号”会过滤掉 neutral，仅展示 strong_long/long/short/strong_short。 Telegram 推送仍以强信号为主（需开启 SIGNAL_PUSH_ENABLED）。

多资产三周期信号监控（Gate 永续）

1D：SMA10 + MACD 背景；4H：EMA20/50 定方向 + ADX(14) 判趋势强度；1H：EMA200 + RSI(14) 触发。1H 与 4H 同向标记“高胜率”，并按 Grade A/B/C 分级。

查看计算规则说明（信号定义 / 准备态 / 高胜率分级 / ATR 风控）
三周期共振逻辑
1D（战略层）：使用 SMA(10) 与 MACD(12,26,9) 判断大背景： bull=Close>SMA10 且 MACD 偏多；bear=Close<SMA10 且 MACD 偏空；其余为 neutral/过渡。若历史不足会显示 —。
4H（战术层）：使用 EMA(20/50) 判方向（EMA20>EMA50 为 up；反之 down）， 用 ADX(14) 判强度（ADX 越大表示趋势越“强”，与上涨/下跌方向无关；常见参考：ADX<20=弱趋势/震荡，ADX≥25=趋势成立（strong），ADX≥40=强趋势/单边更明显）。
1H（执行层）：使用 EMA200 + RSI(14)。 本页面把 1H 分成两种状态：
1H Setup（准备态） vs 1H Signal（触发态）
Setup（准备态）： setup_long=Close>EMA200 且 RSI<30； setup_short=Close<EMA200 且 RSI>70。 该状态表示“接近触发”，用于观察，不会频繁变化。
Signal（触发态，穿越事件）： long=Close>EMA200 且 RSI 从 <30 上穿到 ≥30； short=Close<EMA200 且 RSI 从 >70 下穿到 ≤70。 因为是“事件”，所以大多数时间显示 none 属正常。
高胜率分级（Grade）
当 1H Signal 与 1D 趋势同向时，标记为“高胜率”： Grade A = 同向且 ADX>25（强趋势，优先级最高）； Grade B = 同向但 ADX≤25（趋势不够强，胜率折扣）； Grade C = 非同向或无明确触发（仅作观察）。
风控点位（ATR）
当触发 1H Signal 时： 止损 SL = 1.5×ATR(14)；止盈 TP = 3×ATR(14)（均以 1H ATR 计算）。图表会自动标注 Entry/SL/TP。

策略A（短线）规则
